Nearest timestamp — matches each line to whichever line in the other file starts closest in time. Fast, but can produce crossed or duplicate matches.

Monotonic (DP) — matches lines in time order and never backwards, balancing timing accuracy against leaving lines unmatched. Good default when the files are mostly aligned.

Dynamic Time Warping — allows local stretching/compression to follow drift that grows or shrinks across the file, while still skipping lines that have no match.
